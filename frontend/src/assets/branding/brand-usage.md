# AutoShield Tech — Brand Usage Guide

This document defines how AutoShield Tech branding must be used
across the platform. Follow these rules strictly to maintain a
professional, security-grade appearance.

---

## 1. LOGO VARIANTS

### logo-primary.svg
**Use when:**
- Main dashboards
- Landing pages
- Auth screens
- Marketing visuals
- Trading room backgrounds (hero / featured)

**Notes:**
- This is the authoritative logo
- Prefer centered or top-left placement
- Never stretch or distort

---

### logo-dark.svg
**Use when:**
- Light backgrounds
- White or bright panels
- Reports or exported visuals

---

### logo-light.svg
**Use when:**
- Dark dashboards
- SOC rooms
- Trading panels
- Background imagery overlays

---

### logo-icon.svg
**Use when:**
- Sidebar icons
- Mobile headers
- Favicons
- Compact UI elements

---

### logo-watermark.svg
**Use when:**
- Background imagery
- Rotating security visuals
- Trading room ambient backgrounds
- Dashboard hero sections

**Rules:**
- Opacity: 6%–12%
- Never overpower content
- Decorative only

---

## 2. BACKGROUND IMAGERY RULES

- All background images must feel:
  - Military-grade
  - Cybersecurity-focused
  - Professional (NOT cartoon)
- Logo watermark is allowed and encouraged
- Avoid clutter behind charts or tables
- Motion allowed (slow fades / daily rotation)

---

## 3. COLOR RULES

- Use colors ONLY from `brand-colors.js`
- No hardcoded hex values in components
- Status colors must match:
  - Success → Green
  - Warning → Yellow
  - Danger → Red

---

## 4. PROHIBITED USAGE

❌ Do NOT:
- Modify logo proportions
- Add shadows or glow effects to logos
- Overlay logos on busy imagery
- Use emojis as brand replacements

---

## 5. FUTURE UPGRADES

- Seasonal or campaign visuals must still follow this guide
- New logo variants must be documented here
- Any exception requires explicit approval

---

AutoShield Tech branding represents **trust, control, and vigilance**.
Treat it accordingly.
